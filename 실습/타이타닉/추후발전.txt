## 🔍 Titanic 생존자 예측 모델 추가 실험 방향  

Titanic 생존자 예측 모델을 개선하기 위해, 기존의 최적 조합을 유지하면서 세밀한 최적화를 진행할지, 또는 병렬 처리로 새로운 조합을 탐색할지 고민이 필요하다. 이를 정리하면 다음과 같다.  

---

### 1️⃣ **고정된 최적 조합으로 추가 실험**  
- 기존에 찾은 **최적 데이터셋(A5_C2_E1 등) + 최적 모델(Stacked Ensemble)** 조합을 유지.  
- **Feature Engineering, Hyperparameter Tuning을 적용하여 개선**.  
- 필요하다면 **데이터 증강(SMOTE 등) 후 최적 모델에서만 비교**.  

✅ **장점:**  
✔ 일관된 평가 기준을 유지할 수 있음.  
✔ 불필요한 연산을 줄이고, 최적 조합의 성능을 최대로 끌어올릴 수 있음.  

🚀 **적용해야 할 경우:**  
✔ 현재 최적 조합의 성능이 충분히 높을 때.  
✔ Feature Engineering을 통해 세밀한 성능 개선이 필요할 때.  

---

### 2️⃣ **병렬 처리를 연결하여 새로운 조합 탐색**  
- 기존의 병렬 처리 파이프라인을 활용하여 **최적 모델과 다른 데이터셋 조합도 실험**.  
- 예시:  
  - 최적 모델(Stacked Ensemble) + 모든 데이터셋  
  - 최적 데이터셋(A5_C2_E1) + 모든 모델  
  - Feature Engineering을 추가한 새로운 데이터셋도 병렬로 실험  

✅ **장점:**  
✔ 더 다양한 가능성을 탐색하여 **새로운 최적 조합을 찾을 가능성**이 있음.  
✔ Feature Engineering이 적용된 데이터셋이 있다면 **최적 모델만 유지하는 것보다 데이터 조합을 바꾸어 실험하는 것이 효과적**일 수 있음.  

🚀 **적용해야 할 경우:**  
✔ 기존 최적 조합의 성능이 만족스럽지 않을 때.  
✔ Feature Engineering을 통해 데이터셋이 변경될 때.  
✔ 데이터 증강 기법을 실험하여 추가적인 조합 비교가 필요할 때.  

---

### 📌 **결론: 어떤 방법을 선택할 것인가?**  
✅ **기존 조합의 성능이 충분하다면** → **고정된 조합으로 Hyperparameter Tuning & Feature Engineering**  
✅ **Feature Engineering이 포함된다면** → **병렬 연결하여 새로운 조합도 탐색**  
✅ **컴퓨팅 자원이 부족하다면** → **기존 조합 최적화에 집중**  
✅ **컴퓨팅 자원이 충분하다면** → **새로운 데이터 조합까지 병렬 실험**  

🚀 **추천 방향:**  
🔹 Feature Engineering이 포함된다면 **병렬 확장하여 다양한 조합 실험**  
🔹 Feature Engineering 없이 최적 모델을 개선하려면 **고정된 조합으로 Hyperparameter Tuning**  

---

### 🔜 **향후 실험 계획**  
1️⃣ Feature Engineering 추가 여부 결정  
2️⃣ 고정된 최적 조합으로 Hyperparameter Tuning 수행  
3️⃣ 병렬 확장을 적용할 경우, 새로운 데이터 조합을 반영하여 탐색  
4️⃣ 최종적으로 **고정된 조합 vs 새로운 조합** 비교 후 성능 분석  

➡ **기존 조합이 강력하다면 유지하면서 개선하고, Feature Engineering을 고려한다면 병렬 탐색을 확장하는 것이 최적의 접근법이다.**
